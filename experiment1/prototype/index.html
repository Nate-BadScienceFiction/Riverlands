Re-Run Speculative

The race already happened. Move the sliders to explore alternate timelines. Every change has a cost.

How this works
  • ↓ Less stops — saves clock time, costs Stomach (you ate less) and Morale (you suffered more)
  • ↑ More stops — adds clock time, no cost (more rest)
  • ↑ Higher effort — saves clock time, costs Legs (eccentric load) and Fuel (metabolic depletion)
  • ↓ Lower effort — adds clock time, no cost (easier pace)

Costs are banded: small adjustments are cheap, big ones are cliff-expensive. Any pool reaching 0 = DNF. Speculative replay — cost-on-arithmetic, not physiology, not training advice.

    GitHub Pages isn't running an application server — it's a static file host backed by a CDN. On every push to main, an Actions workflow bundles the experiment1/prototype/ directory into a "Pages artifact" and hands it to GitHub's Pages deployment service, which unpacks it onto edge servers worldwide. When a visitor hits the URL, GitHub's CDN serves the raw files — index.html, app.js, styles.css, the vendored libraries (Leaflet, Chart.js, suncalc), the bundled Riverlands.gpx, and the ~174 pre-cached map tile PNGs — over HTTPS. The application itself runs entirely in the visitor's browser: it loads index.html, executes the JavaScript locally, and uses fetch() against the same Pages origin to pull the GPX and any additional tiles. GitHub never executes any of your code — it just serves bytes. The Python server.py and serve.bat exist only for local development; in production, GitHub's CDN plays the same role any plain static HTTP server would.
